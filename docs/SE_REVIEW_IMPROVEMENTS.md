# 上位レビューによる改善点

## 今回入れた改善
1. 同じ案内を連続で受けた時は、2回目以降を短くする
2. 具体的な相談は入口helpで止めず、本体へ流しやすくする
3. 痛み相談を homecare より先に扱う
4. `buildPainReply` 内の重複条件を修正する

## 利用者にとっての意味
- 同じ説明が何度も長く返らず、会話の負荷が下がる
- 「腰が固まりやすくて家で何したらいい？」のような具体相談が、
  入口helpに吸われにくくなる
- 痛みがある相談は、まず症状整理寄りで受け止めやすくなる
- 小さな実装不具合を潰して、安定性を上げる

## ここから。の思想との整合
- 入口で説明しすぎない
- 相談の具体性を拾って、必要以上に戻さない
- 痛みやしびれなど状態優先のときは、ケア提案より先に受け止める
- 同じ案内を繰り返して疲れさせない
